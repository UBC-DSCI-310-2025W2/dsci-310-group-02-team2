# Contributing to This Project

Thank you for your interest in contributing to this project!

We welcome all contributions that help improve the project, including bug fixes, feature suggestions, documentation improvements, and code enhancements.

---

## Reporting Issues

If you encounter a bug, unexpected behavior, or have suggestions for improvements, please open an issue on our GitHub repository.

You can submit issues here:

https://github.com/UBC-DSCI-310-2025W2/dsci-310-group-02-team2
/issues

When creating an issue, please include:

- A clear and descriptive title
- A detailed explanation of the problem or suggestion
- Steps to reproduce the issue (if applicable)
- Any relevant screenshots or error messages

---

## Contributing Code

If you would like to contribute code improvements or new features:

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes
4. Commit your changes with clear commit messages
5. Submit a pull request to the main repository

Example workflow:

```
git fork
git checkout -b feature-name
git commit -m "Add new feature"
git push origin feature-name
```

Once your pull request is submitted, the project maintainers will review it.

---

## Pull Request Guidelines

To help us review contributions efficiently, please ensure that:

- Your code is clear and well-documented
- Your changes do not break existing functionality
- Your commit messages clearly describe the changes made
- Your pull request includes a description of what was changed and why

Pull requests will typically be reviewed within **7 days**.
